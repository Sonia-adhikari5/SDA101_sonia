This practical focused on creating a complete HTML form with styling and validation, simulating a real-world use case for student hostel maintenance requests. It was a highly practical exercise that combined structure, validation, styling, and responsive design.

## New Skills and Knowledge Acquired

Through this assignment, I learned how to build well-organized forms that are both functional and visually appealing. Key takeaways included:

Structuring a form into logical sections improves readability and usability.

Implementing HTML input attributes like required, minlength, maxlength, type, and placeholder helps prevent user errors and enforces validation at the browser level.

Using radio buttons and dropdowns effectively allows for controlled input while maintaining user choice flexibility.

I gained hands-on experience with CSS styling techniques such as box shadows, border-radius, colors, focus states, and padding, making the form modern and accessible.

Learned to combine Flexbox and Grid layout systems to create responsive designs, ensuring the form works across multiple devices.

This practical also reinforced the importance of user experience (UX) when designing forms. Every input field, label, and section was considered carefully to guide the user through the submission process with minimal confusion.

## Challenges Faced and How They Were Overcome

One challenge was ensuring proper validation for all inputs. Some fields, like the student ID and issue description, required specific length constraints. Initially, I was unsure how to enforce these constraints without relying on JavaScript. I overcame this by exploring HTML attributes like minlength, maxlength, and required, which effectively validated input at the browser level.

Another challenge was designing a form that looked professional and modern. Adding box shadows, border-radius, and consistent spacing required experimenting with CSS properties and testing on different screen sizes. To solve this, I iteratively adjusted styles, focusing on readability, alignment, and mobile responsiveness.

Lastly, creating a responsive layout was slightly tricky. Some input fields did not stack properly on smaller screens at first. Using CSS Grid and Flexbox allowed me to ensure that the form and its elements adapt gracefully to mobile devices.
