# Reflection on Practical 7

## What I Learned
Practical 7 gave me hands-on experience with **TailwindCSS**, which is very different from traditional CSS because it relies on utility classes rather than writing custom CSS rules. This allowed me to create layouts quickly and efficiently, without worrying about naming classes or creating separate style sheets.  

I learned to use **Flexbox** utilities for navigation bars, making it easier to align items horizontally and manage spacing. Similarly, I used **Grid utilities** to create a responsive photo gallery that adapts seamlessly to different screen sizes. Tailwind's responsive prefixes, like `sm:`, `md:`, and `lg:`, made it simple to define layouts for mobile, tablet, and desktop without writing media queries manually.

Adding hover effects, transitions, and scale animations helped me understand how micro-interactions improve user experience. I also realized how Tailwind promotes consistency in spacing, colors, and typography across the project, which is essential for professional-looking web design.

## Challenges Faced
1. **Responsive Design:** Initially, I struggled with making the navigation bar and gallery fully responsive. Elements would overlap on smaller screens, and the layout looked unbalanced.  
2. **Tailwind Utility Classes:** Since Tailwind uses many short utility classes, I sometimes found it hard to remember all of them and their exact syntax.  
3. **Combining Multiple Layouts:** Implementing both Flexbox and Grid in the same project was challenging, especially aligning the sections while keeping a consistent style.

- I referred to Tailwind documentation frequently to find correct class names and understand how responsive prefixes work.

- For layout issues, I experimented by combining `flex`, `flex-wrap`, `justify-between`, and `grid-cols-*` utilities until the sections aligned correctly.  

## Overall Reflection
This practical gave me a strong understanding of **modern front-end development practices**. I now feel confident using TailwindCSS to build **responsive, interactive, and visually appealing websites**. It also helped me appreciate the efficiency of utility-first frameworks over traditional CSS in rapid web development.  

I plan to explore Tailwind further by building more complex components, like modals and carousels, and integrating Tailwind with JavaScript frameworks like React to enhance interactivity. This practical reinforced the importance of responsive design and clean layouts, which are crucial for professional web development.
